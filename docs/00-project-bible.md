# Club Zilla Digital Platform — Project Bible

## Document purpose
This project bible is the central reference for the Club Zilla Digital Platform during discovery and requirements definition. It defines shared terminology, known project facts, documentation rules, and open questions.

## Project identity
- **Project name:** Club Zilla Digital Platform
- **Client:** Club Zilla School
- **Development company:** Appartisans Lab
- **Current phase:** Discovery and requirements definition

## Current phase objective
The objective of the current phase is to define and validate the MVP, Business Requirements Document (BRD), Software Requirements Specification (SRS), and architecture before production development begins.

## Source of truth hierarchy
1. Approved client decisions
2. Approved MVP scope
3. Approved BRD
4. Approved SRS
5. Approved architecture
6. Repository documentation
7. Working assumptions marked as `TBD` or `Needs clarification`

## Documentation principles
- Do not invent requirements.
- Mark unknown information as `TBD`.
- Label business rules as `Proposed`, `Approved`, `Rejected`, or `Needs clarification`.
- Keep terminology consistent across all documents.
- Do not create backend, frontend, mobile, infrastructure, or test code during discovery.

## Known scope
- Create a documentation foundation for the Club Zilla Digital Platform.
- Capture product vision, user roles, and business rules at a discovery-stage level.
- Preserve the existing `.gitkeep` file.

## Out of scope for current phase
- Backend code
- Frontend code
- Mobile code
- Infrastructure code
- Test code
- Production development
