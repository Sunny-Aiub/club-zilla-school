# AGENTS.md

## Scope
These instructions apply to the entire Club Zilla Digital Platform repository.

## Project context
- Project name: Club Zilla Digital Platform
- Client: Club Zilla School
- Development company: Appartisans Lab
- Current phase: Discovery and requirements definition

## Agent instructions
- Do not invent requirements, features, integrations, policies, timelines, user roles, or technical decisions.
- Mark unknown, missing, or unconfirmed information as `TBD`.
- Keep terminology consistent across all documentation.
- Do not start production development until the MVP, Business Requirements Document (BRD), Software Requirements Specification (SRS), and architecture are approved.
- Do not create backend, frontend, mobile, infrastructure, or test code during the discovery and requirements definition phase unless explicitly authorized after approval.
- Preserve the existing `.gitkeep` file.
- Clearly label business rules as `Proposed`, `Approved`, `Rejected`, or `Needs clarification`.
- Prefer documentation changes over implementation changes while the project remains in discovery.
